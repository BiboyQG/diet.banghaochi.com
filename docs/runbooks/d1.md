# D1 Runbook

## Local

```bash
npm run db:migrate:local
npm run dev:worker
```

Check local state:

```bash
npx wrangler d1 execute diet-tracker --local --command "SELECT * FROM profile"
npx wrangler d1 execute diet-tracker --local --command "SELECT * FROM daily_targets"
npx wrangler d1 execute diet-tracker --local --command "SELECT name, calories_kcal, carbs_g, protein_g, fat_g FROM food_templates WHERE deleted_at IS NULL"
```

The `0002_food_templates.sql` migration creates the reusable common foods table and seeds the default Chipotle bowl template.

## Staging

The staging database is `diet-tracker-staging`. Its binding ID is configured in `apps/worker/wrangler.jsonc`.

```bash
npx wrangler d1 migrations apply DB --env staging --remote
npx wrangler deploy --env staging
```

## Production

The production database is `diet-tracker-production`. Its binding ID is configured in `apps/worker/wrangler.jsonc`.

```bash
npx wrangler d1 migrations apply DB --env production --remote
npx wrangler deploy --env production
```

## Export

```bash
npx wrangler d1 export diet-tracker-production --remote --output backup.sql
```
